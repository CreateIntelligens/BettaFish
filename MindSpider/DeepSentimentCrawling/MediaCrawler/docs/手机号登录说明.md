# 關於手機號+驗證碼登錄的說明
> 配置過程相當複雜，不建議採用該種方式

當在瀏覽器模擬人爲發起手機號登錄請求時，使用短信轉發軟件將驗證碼發送至爬蟲端回填，完成自動登錄

準備工作：

- 安卓機1臺（IOS沒去研究，理論上監控短信也是可行的）
- 安裝短信轉發軟件 [參考倉庫](https://github.com/pppscn/SmsForwarder)
- 轉發軟件中配置WEBHOOK相關的信息，主要分爲 消息模板（請查看本項目中的recv_sms_notification.py）、一個能push短信通知的API地址
- push的API地址一般是需要綁定一個域名的（當然也可以是內網的IP地址），我用的是內網穿透方式，會有一個免費的域名綁定到內網的web
  server，內網穿透工具 [ngrok](https://ngrok.com/docs/)
- 安裝redis並設置一個密碼 [redis安裝](https://www.cnblogs.com/hunanzp/p/12304622.html)
- 執行 `python recv_sms_notification.py` 等待短信轉發器發送HTTP通知
- 執行手機號登錄的爬蟲程序 `python main.py --platform xhs --lt phone`
